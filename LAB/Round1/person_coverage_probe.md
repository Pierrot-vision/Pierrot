# 인물 학습셋의 빈 구멍 — 커버리지 프로브

작성일: 2026-07-20

PIERROT의 전신 인물 학습셋(`person_full_model`, 177,357장)이 **무엇을 담고 있고 무엇을 담고 있지 않은지**를 캡션 통계로 확인한 뒤, **담고 있지 않은 것만 골라** 프롬프트를 만들어 1.6B 모델에 물어본 기록이다.

목적은 품질 자랑이 아니라 진단이다. 학습셋에 없는 조건을 요구했을 때 모델이 **일반화해서 그려내는지, 아니면 학습셋 분포로 되돌아가는지**를 보고, 2라운드 데이터 설계의 근거로 삼는다.

## 1. 학습셋에 무엇이 들어 있나

캡션 6만 건(18개 parquet 중 6개)을 어휘 단위로 세었다. 캡션은 평균 324자이며, 다음과 같은 고정된 틀을 따른다.

> `a slim white woman with short wavy hair is wearing a slim light-gray rib-knit plain long sleeve top with hip-length and a turtle neckline, regular long light-gray cotton-linen stripe trousers with a wide silhouette, black leather sandal with a open-toe ...`

체형 → 인종 → 머리 → 상의(핏·색·소재·패턴·기장·넥라인) → 하의 → 신발 → 액세서리 순서가 거의 그대로 반복된다. **표현이 다양한 것이 아니라 슬롯만 바뀌는 구조**다.

### 1.1 체형

| 표현 | 비율 |
| --- | --- |
| slim | 65.0% |
| average-sized | 43.3% |
| muscular | 4.0% |
| **plus-size / curvy / petite / athletic / chubby / overweight** | **0%** |

마른 체형과 보통 체형이 사실상 전부다. **큰 체형, 굴곡 있는 체형, 아주 작은 체형은 한 건도 없다.**

### 1.2 인종

| 표현 | 비율 |
| --- | --- |
| white | 73.4% |
| black | 67.8% |
| hispanic / latino | 13.5% |
| asian | 12.2% |
| **south asian / indian / middle-eastern** | **0%** |

백인과 흑인이 대부분이고, **남아시아·중동은 완전히 비어 있다.** 아시아도 12%로 낮다.

### 1.3 착장 스타일 (tuck / fold)

| 표현 | 비율 |
| --- | --- |
| untucked | 63.0% |
| open | 22.8% |
| tucked | 16.1% |
| rolled | 7.4% |
| layered | 0.01% |
| **half-tucked / folded / cuffed** | **0%** |

넣거나(tucked) 빼거나(untucked) 둘 중 하나뿐이다. **앞만 넣기(half-tuck), 소매·밑단 접기(cuff), 깃 세우기(fold)** 같은 중간 상태는 없다.

### 1.4 의류 종류

| 표현 | 비율 |
| --- | --- |
| shirt | 50.0% |
| t-shirt | 34.9% |
| jeans | 28.6% |
| pants / trousers | 27% |
| jacket | 8.9% |
| dress | 8.4% |
| coat | 1.2% |
| suit | 0.2% |
| **shorts** | **0.005% (3건)** |

셔츠와 청바지 중심의 캐주얼 정장 조합에 몰려 있다. **반바지는 6만 건 중 3건**, 정장은 0.2%다.

## 2. 어떻게 물어봤나

위에서 확인한 **0%이거나 극히 희소한 축만 골라** 24개 프롬프트를 만들었다. 전문은 [pfm_probe_prompts.json](../pfm_probe_prompts.json)에 있다.

## 4. 관찰

### 4.1 되는 것 — 학습셋에 없어도 그려낸다

**인종은 학습셋 공백에도 불구하고 잘 나온다.** 남아시아 여성(id 7)의 짙은 갈색 피부와 검은 머리, 인도 남성(id 8)의 쿠르타, 중동 남성(id 9)의 짧은 수염, 히잡 착용 여성(id 10) 모두 요청대로 그려졌다. 학습셋에 `indian` · `middle-eastern` 이라는 단어가 **한 건도 없었는데도** 나온 것이다.

이는 인물 학습셋이 전부가 아니라는 뜻이다. base 학습 전체에는 다양한 인종의 사진이 들어 있고, 텍스트 인코더(Qwen3-4B)도 이 개념을 알고 있다. **인물 데이터셋의 공백을 다른 데이터가 메워 준 것**으로 보인다.

**의류 종류도 마찬가지다.** 반바지(id 19, 학습셋 3건), 정장(id 18, 0.2%), 롱코트(id 20, 1.2%) 모두 형태가 정확하다. 특히 핀스트라이프 더블 브레스트 정장은 라펠과 단추 배치까지 맞다.

**다인종 2인 구성(id 12)** 도 성공했다. 흑인 여성의 머스터드 점프수트와 동아시아 여성의 흰 블라우스·남색 스커트가 각각 정확히 배치됐다 — 앞서 [vs_prx.md](vs_prx.md)에서 확인한 속성 결합 강점과 일치한다.

### 4.2 안 되는 것 — 체형은 학습 분포로 되돌아간다

**가장 뚜렷한 실패는 체형이다.**

- **plus-size 여성(id 1)** — 요청보다 훨씬 마르게 나왔다. "plus-size"라고 명시했는데 결과는 보통 체형에 가깝다.
- **petite 여성(id 3)** — "작은 체구, 낮은 키"를 요청했으나 일반적인 모델 체형으로 그려졌다. 트렌치코트가 발목까지 온다는 묘사만 반영됐다.
- **curvy 여성(id 4)** — 모래시계 체형 요청이 반영되지 않았다.
- **tall lanky 남성(id 6)** — 요청한 마르고 긴 체형 대신 보통 체격이 나왔다.

성공한 것은 **heavyset 남성(id 2)** 과 **athletic 여성(id 5)** 정도다. id 2는 코트가 부피를 가려 판단이 어렵고, id 5는 운동복이라는 맥락이 체형을 도왔을 가능성이 있다.

**이 실패 방식이 인종과 정반대라는 점이 중요하다.** 인종은 데이터셋에 없어도 다른 곳에서 배웠지만, 체형은 되돌아갔다. 웹 이미지 전반이 마른 체형에 치우쳐 있어 **다른 데이터도 같은 편향을 공유**하기 때문으로 보인다. 인물 데이터셋만의 문제가 아니라는 뜻이다.

### 4.3 애매한 것 — 미세한 착장 상태

**half-tuck(앞만 넣기)은 절반만 된다.** id 13은 앞자락이 청바지에 들어가 있어 성공에 가깝지만, id 14는 일반적인 언턱 상태로 나왔다. "앞만 넣고 뒤는 빼라"는 지시의 **부분성**이 잘 전달되지 않는다.

**소매·밑단 접기(cuff)는 된다.** id 15는 소매가 팔꿈치까지 걷어올려져 있다. 학습셋에 `rolled`가 7.4% 있어서 그 개념이 전이된 것으로 보인다.

**깃 세우기(fold)는 안 된다.** id 16의 트렌치코트는 깃이 세워지지 않고 평범하게 눕혀져 있다. 벨트를 묶은 것만 반영됐다.

**3겹 레이어(id 17)** 는 겉의 카디건과 바지만 보이고 안쪽 티셔츠 층이 표현되지 않았다. 층수를 세는 요청은 아직 어렵다.

### 4.4 속성 조합

소재 대비(id 21 — 케이블니트 + 가죽 바지), 패턴 혼합(id 22 — 플로럴 블라우스 + 체크 스커트), 색 블로킹(id 23 — 주황 패딩 + 라임 팬츠)은 **모두 요청대로 나왔다.** 뒷모습(id 24)도 성공했다.

## 5. 정리 — 2라운드 데이터에 주는 함의

이 프로브가 알려주는 것은 **"어떤 공백이 메워지고 어떤 공백은 안 메워지는가"** 다.

| 축 | 학습셋 | 결과 | 해석 |
| --- | --- | --- | --- |
| 인종 | 0% | ✅ 됨 | 다른 데이터가 메워 줌 |
| 의류 종류 | 0.005~1.2% | ✅ 됨 | 다른 데이터가 메워 줌 |
| 속성 조합 | 희소 | ✅ 됨 | 조합 능력은 이미 있음 |
| **체형** | 0% | ❌ **안 됨** | **전체 데이터가 같은 편향 공유** |
| 미세 착장(half-tuck·fold) | 0% | 🔶 절반 | 인접 개념(rolled)이 있으면 전이됨 |

여기서 나오는 결론은 세 가지다.

**첫째, 특정 데이터셋의 공백만 보고 판단하면 안 된다.** 인종은 `person_full_model`에 0%였지만 실제로는 잘 나왔다. 데이터 보강을 계획할 때는 **전체 학습셋을 합쳐서** 무엇이 부족한지 봐야 한다.

**둘째, 체형은 의도적으로 채워야 한다.** 웹에서 긁어모으는 방식으로는 마른 체형 편향이 그대로 따라온다. [round1_experiment_report.md](round1_experiment_report.md) 6절 ④에서 "부족한 종류를 지정해 생성한다"고 적은 항목에 **체형을 명시적으로 포함**해야 한다.

**셋째, 인접 개념이 있으면 전이된다.** `rolled`가 7.4% 있었기에 `cuffed`가 됐고, `fold`는 인접 개념이 없어 실패했다. 새 개념을 넣을 때 **0에서 시작하지 말고 비슷한 것부터 깔아 두는 것**이 효율적이다.

한계도 분명하다. **프롬프트당 1장, seed 1개**이므로 위 판정은 확정이 아니다. 특히 체형은 seed를 바꾸면 결과가 달라질 수 있어, 실제 데이터 계획에 반영하기 전에 체형 축만 여러 seed로 반복 측정할 필요가 있다.
