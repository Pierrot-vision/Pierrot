# PIERROT 1차 실험 완료 보고 — 전체 회고와 2차 실험의 근거

작성일: 2026-07-20

이 문서는 PIERROT의 **1차 실험이 GPU 종료로 마감된 시점**에서, 지금까지 돌린 모든 학습 트랙을 한자리에 모아 정리한 것이다. 목적은 두 가지다.

1. **무엇을 실제로 했고, 무엇이 검증됐고, 무엇이 검증되지 않은 채로 끝났는가**를 사실 그대로 남긴다.
2. **2차 실험을 설계할 때 근거로 쓸 수 있게** 한다. GPU가 다시 확보됐을 때 "어디서부터, 무엇을 바꿔서" 시작할지의 판단 재료다.

기존 LAB 문서들이 **한 모델의 step별 관찰**을 담았다면([0.8b_training_review.md](0.8b_training_review.md) / [1.6b_training_review.md](1.6b_training_review.md) / [vs_prx.md](vs_prx.md)), 이 문서는 **트랙 전체를 가로지르는 결산**이다. 개별 샘플 관찰은 반복하지 않고, 그 문서들을 가리킨다.

> **1차 실험의 종료 사유는 목표 달성이 아니라 자원 소진이다.** 아래 phase3(1.6B)와 phase4(3B)는 학습 곡선이 아직 살아 있는 상태에서, 계획된 종료 지점이 아니라 GPU 사용 종료 시점에서 멈췄다. 이 점이 모든 결론의 전제다.

## 1. 1차 실험이란 무엇이었나

PIERROT는 텍스트로 이미지를 만드는 확산 모델(diffusion model, 노이즈를 조금씩 걷어내며 그림을 만드는 방식)을 **처음부터 직접 학습**하는 프로젝트다. 1차 실험은 그 첫 사이클이며, 다음 질문에 답하는 것이 목표였다.

| # | 질문 | 답 여부 |
| --- | --- | --- |
| Q1 | 이 구조로 **말이 되는 그림이 나오는가** | ✅ 답함 |
| Q2 | 큰 선생 모델을 베껴 출발점을 만드는 방식(**지식 증류, KD**)이 도움이 되는가 | ✅ 답함 (도움 안 됨) |
| Q3 | 작은 모델을 키워서 이어 학습하는 **깊이 성장(depth growth)** 이 실제로 작동하는가 | ✅ 답함 (작동함) |
| Q4 | 같은 방식으로 **3B까지 올려도 유지되는가** | ❌ 미답 (평가 전 종료) |
| Q5 | **외부 공개 모델과 견줄 만한가** | 🔶 부분 답함 (정성 비교만, 수치 없음) |
| Q6 | 후처리 학습(**SFT**)으로 품질을 끌어올릴 수 있는가 | ❌ 미답 (원인 진단까지만) |

Q1~Q3은 답을 얻었고, Q4~Q6은 열린 채로 남았다. 2차 실험의 출발점은 여기다.

## 2. 실험 지도 — 다섯 트랙이 어떻게 이어졌나

트랙은 시간순이 아니라 **가중치가 흘러간 순서**로 이어져 있다. 아래 화살표는 "이전 트랙의 가중치를 받아 이어 학습했다"는 뜻이다.

```
  [phase1] KD pretrain 0.8B ────✗ 폐기 (가중치 인계 안 함)
                                   │
  랜덤 초기화 ──────────────────────┴──> [phase2] 0.8B base
                                              │ 2.745M step
                            ┌─────────────────┼─────────────────┐
                            │                 │                 │
                     [phase2_sft]      [phase3] 1.6B      (phase2 종료)
                     0.8B SFT          깊이 16→33장             │
                     55k step          930k step                │
                     ✗ 화질 저하             │                  │
                                             └──> [phase4] 3B ──┘
                                                  깊이 33→64장
                                                  305k step
                                                  ⚠ 평가 전 종료
```

시간대는 다음과 같다. 체크포인트 파일의 실제 기록 시각 기준이며, 초기 체크포인트는 디스크 확보를 위해 지운 것이 있어 **관측 가능한 구간만** 적는다.

| 트랙 | 관측 시작 | 관측 종료 | 기간 | 상태 |
| --- | --- | --- | --- | --- |
| phase1 (KD 0.8B) | 2026-05-25 | 2026-05-28 | 약 2.4일 | ✗ 폐기 |
| phase2 (0.8B base) | 2026-05-15 (step 0.5M) | 2026-07-10 (step 2.745M) | 약 56일 | ⏹ 종료 |
| phase2_sft / sft_2 | 2026-06-26 | 2026-07-01 | 약 5일 | ✗ 중단 |
| phase3 (1.6B) | 2026-06-18 | 2026-07-18 | 약 30일 | ⏹ GPU 종료로 정지 |
| phase4 (3B) | 2026-07-10 | 2026-07-20 | 약 10일 | ⏹ GPU 종료로 정지 |

phase3와 phase4의 기간이 phase2와 겹친다. 노드가 두 대(looko1 / looko2)이므로 **두 트랙이 동시에 돌았다**고 읽는 것이 자연스럽다. 실제로 phase4는 phase2가 끝난 날(07-10)에 시작했다 — 자리가 비자마자 다음 체급이 들어갔다는 뜻이다.

## 3. 트랙별 결과

### 3.1 phase1 — KD pretrain (실패로 확정)

큰 모델(FLUX.2-klein-base-4B, 3.876B)을 **선생**으로 두고, 작은 PIERROT(0.857B)가 그 출력을 따라 하도록 학습해 좋은 출발점을 만들려는 시도였다. 기대는 "+2~5% 품질"이었고, 그 수치는 **추정치일 뿐 측정된 적이 없다**고 계획 문서에 이미 적혀 있었다.

| 항목 | 값 |
| --- | --- |
| 학생 / 선생 | 0.857B / FLUX.2-klein-base-4B 3.876B (bf16, 고정) |
| 로그 관측 구간 | step 90,010 → 129,050 (약 39,000 step) |
| 배치 | GPU당 1 × 8 GPU = 8 |
| 남은 체크포인트 | 12개 (10k ~ 120k), 115 GB |
| 가속 기법 | 전부 OFF (순수 KD 신호만) |

**판정 근거는 손실(loss) 곡선이 움직이지 않았다는 것이다.** 39,000 step 동안 KD 손실은 최소 1.4067, 최대 2.3172, 평균 1.8166으로 **아래로 내려가는 추세가 없었다.**

| 지점 | step | KD loss |
| --- | --- | --- |
| 시작 | 90,010 | 1.8405 |
| 25% | 99,760 | 1.6779 |
| 50% | 109,520 | 1.8709 |
| 75% | 119,280 | 1.7236 |
| 끝 | 129,050 | 1.8230 |

학습이 진행되면 손실은 내려가야 한다. 위 표는 **오르내리기만 할 뿐 내려가지 않는다**. 3.9만 step을 더 돌려서도 방향이 잡히지 않았다는 것은, 이 설정으로는 선생의 지식이 학생에게 옮겨가지 않고 있다는 뜻이다.

결과적으로 **phase2 base 학습은 KD 가중치를 물려받지 않고 랜덤 초기화에서 시작했다.** 즉 phase1은 최종 모델 계보에 기여하지 않았고, 115 GB의 체크포인트와 약 2.4일의 GPU 시간이 회수되지 않은 비용으로 남았다.

> **2차에 주는 교훈**: KD는 "붙여 보면 좋겠지" 수준으로 들어갈 기법이 아니다. 선생과 학생의 노이즈 스케줄·시간축 정의가 정확히 맞아야 신호가 전달되는데, 그 정합을 **손실 곡선이 내려가는지로 조기에 판정**했어야 한다. 39,000 step은 판정에 필요한 양보다 훨씬 많다. 2차에서 KD를 다시 시도한다면 **5,000 step 안에 하락 추세가 보이지 않으면 즉시 중단**하는 규칙을 먼저 정해야 한다.

### 3.2 phase2 — 0.8B base (1차 실험의 본체)

가장 오래, 가장 많이 돌린 트랙이다. 랜덤 초기화에서 시작해 **2,745,000 step**까지 갔다.

| 항목 | 값 |
| --- | --- |
| 파라미터 | 857.07M (0.857B), 레이어 16장 |
| 최종 step | 2,745,000 |
| 배치 (최종 구간) | GPU당 12 × 8 GPU = 96 |
| 학습률 | 1.0e-4 (warmup 1000 후 상수) |
| 체크포인트 | 74개, 500k ~ 2.745M, **905 GB** |
| 보조 손실 | LPIPS (vgg, 0.1) + P-DINO (dinov2_vitb14_reg, 0.01) |
| 데이터 | 36개 데이터셋 혼합, 1024px, 40개 화면비 버킷 |
| 최종 구간 step 시간 | 6.13초 |

**품질 궤적은 "깨끗 → 회화풍으로 물듦 → 회복"의 U자**를 그렸다. 상세 관찰은 [0.8b_training_review.md](0.8b_training_review.md)에 있고, 요약하면 1.0M이 가장 중립적이었고 2.0M에서 유화풍이 정점을 찍었다가 2.5M~2.745M에서 사실적 사진톤으로 돌아왔다. **최종 2.745M이 이 트랙의 최상 지점이다.**

수치 평가는 딱 한 번 있었다. phase2 step 2,695,000에 대한 GenEval(생성 모델의 지시 이행 능력을 자동 채점하는 벤치마크) 결과다.

| 항목 | 점수 | 참고 |
| --- | --- | --- |
| **overall** | **0.4416** | SD1.5 약 0.43 / SDXL 약 0.55 / Flux.1-dev 약 0.66 |
| single_object | 0.9313 | |
| colors | 0.6809 | |
| two_object | 0.5101 | |
| counting | 0.2750 | 개수 세기 — 정성 관찰과 일치 |
| color_attr | 0.1300 | |
| position | 0.1225 | |

다만 이 0.4416은 **그대로 믿기 어렵다.** 실패 사례 1,267건 중 918건(72%)이 "채점용 물체 검출기가 아무것도 못 찾음"이었고, 학습 때 켜져 있던 프롬프트 확장 설정(chi_prompt)이 채점 생성 때는 꺼져 있었다. 즉 **모델 품질보다 채점 파이프라인 쪽 문제가 점수를 끌어내렸을 가능성이 크다.** 이 재평가는 하지 못한 채 1차가 끝났다.

### 3.3 phase2_sft — SFT 시도 (중단, 원인은 진단됨)

base 모델을 고품질·정제 데이터로 다듬어 화질을 올리려는 시도였다. 두 번 돌렸고 둘 다 목표에 도달하지 못했다.

| 회차 | 기간 | 최종 step | 결과 |
| --- | --- | --- | --- |
| phase2_sft (1차) | 06-26 ~ 06-27 | 17,500 | 화질 저하 → 체크포인트 167 GB 삭제 |
| phase2_sft_2 (2차) | 06-27 ~ 07-01 | 55,000 | 조건 바꿔 재시작, 판정 전 중단 |

**증상은 "의미는 멀쩡한데 선명도가 점점 깎인다"** 였다. 진단된 원인은 두 가지다.

1. **화질을 지켜 주던 보조 손실이 꺼져 있었다.** base 학습에는 LPIPS(사람 눈이 느끼는 차이를 재는 손실)와 P-DINO가 켜져 있었는데, 1차 SFT에서는 둘 다 꺼져 있었다. 남은 것은 픽셀 차이만 재는 MSE뿐이고, MSE만으로 학습하면 **평균값으로 수렴하려는 성질 때문에 그림이 뿌예진다.** 애매하면 흐릿하게 그리는 쪽이 평균 오차가 작기 때문이다.
2. **데이터가 합성 이미지 쪽으로 쏠렸다.** SFT 데이터 혼합의 약 63%가 합성이었다. 합성 이미지 특유의 매끈함을 계속 학습하면 실사 질감이 빠진다.

2차 SFT는 LPIPS·P-DINO를 켜고 학습률을 3e-5로 낮춰 재시작했지만, **55,000 step에서 판정 전에 멈췄다.** 이 트랙은 원인 가설까지만 세워 두고 검증하지 못한 상태다. 실험 일지는 [SFT.md](SFT.md)에 있다.

> **한 가지 더 짚어 둘 점**: LPIPS와 P-DINO는 실제로는 VAE가 압축한 잠재(latent) 텐서의 앞 3채널을 가짜 RGB처럼 취급해 계산됐다. 진짜 픽셀 위에서 재는 화질 손실이 아니라 **거친 근사**다. 화질이 핵심 목표라면 2차에서는 이 부분을 손봐야 한다.

### 3.4 phase3 — 1.6B, 깊이 성장 (1차 실험의 최대 성과)

0.8B(레이어 16장)의 가중치를 물려받아 **레이어를 33장으로 늘린 뒤 이어서 학습**했다. 큰 모델을 처음부터 학습하는 대신 이미 수렴한 작은 모델을 출발점으로 재사용하는 방식(SANA 1.5의 efficient model growth)이며, GPU 예산이 빠듯한 환경에서 사실상 유일하게 현실적인 확장 경로다.

| 항목 | 값 |
| --- | --- |
| 파라미터 | 1,694.74M (1.695B), 레이어 33장 |
| 최종 step | 930,000 |
| 배치 | GPU당 5 × 8 GPU = 40 |
| 체크포인트 | 68개, 5k ~ 930k, **1.3 TB** |
| 보조 손실 | REPA (dinov2_vitl14_reg, layer 7, λ=0.5) + LPIPS + P-DINO |
| 최종 구간 step 시간 | 2.11초 |
| 최종 구간 평균 손실 | **1.2044** — 다섯 트랙 중 가장 낮음 |
| 추가 구조 | step 330k에서 텍스트 어댑터 도입 (+77M) |

**깊이 성장이 작동한다는 것이 이 트랙의 핵심 결과다.** 레이어를 늘린 직후(5k)에는 60장의 샘플이 거의 전부 유화처럼 무너졌지만, **5만 step 만에 대부분 사실적 사진톤으로 돌아왔다.** 붕괴한 것은 지식 전체가 아니라 표현의 배치였고, "무엇을 그릴지"에 대한 0.8B의 지식은 살아 있었다는 뜻이다.

부수적으로 **텍스트 어댑터가 품질을 깨지 않는다**는 것도 확인됐다. 도입 직전(300k)과 직후(350k)의 샘플이 사실상 동등했다. 어댑터는 0으로 초기화되어 들어오므로 이론과도 일치한다. "phase3 품질 저하가 어댑터 탓"이라는 의심은 이 비교로 기각됐다.

가장 눈에 띄는 능력 향상은 **이미지 안에 글자 쓰기**였다. 체급이 올라가며 네 단계를 밟았다.

| 체크포인트 | "MORNING LOAF" | "DEEP WORK" |
| --- | --- | --- |
| 0.8B 2.50M | `MORNIING IOODINE` — 판독 불가 | `DEEPTN / RD EPFFROWORK` — 판독 불가 |
| 0.8B 2.60M | `MORNING TOORLOFF` — 앞 단어만 | `DEEPEK / DEEPEKOWOREK` — 판독 불가 |
| 0.8B 2.745M | `MORNING LOAFF` — F 중복 | `DEEP WORRK` — R 중복 |
| 1.6B 800k | `MORNING LOAF` — **정확** | `DEEP WORK` — **정확** |
| 1.6B 850k | `MORNING LOAF` — **정확** | `DEEP WORK` — **정확** |
| 1.6B 930k | `MORING LOAF` — N 누락 | `DEEP WORK` — 정확 |

step별 상세 관찰과 104개 진단 프롬프트 결과는 [1.6b_training_review.md](1.6b_training_review.md)에 있다.

### 3.5 phase4 — 3B, 두 번째 깊이 성장 (평가 전 종료)

1.6B(레이어 33장)를 다시 **레이어 64장**으로 키운 트랙이다. 1차 실험에서 **가장 아쉬운 자리**다.

| 항목 | 값 |
| --- | --- |
| 파라미터 | 3,081.66M (3.082B), 레이어 64장 |
| 최종 step | **305,000** (2026-07-20 12:17 기록) |
| 배치 | GPU당 3 × 8 GPU = 24 |
| 체크포인트 | 27개 이상, 2.5k ~ 305k, **907 GB** |
| 보조 손실 | LPIPS + P-DINO (**REPA 없음**) |
| TREAD 라우팅 | prob 0.5, 블록 [2, 48) — 하위 체급의 [2, 12)보다 훨씬 넓음 |
| 최종 구간 step 시간 | 1.68초 |
| 최종 구간 평균 손실 | 1.6406 — 49,000 step 동안 **평평하거나 소폭 상승** |

**문제는 이 트랙을 눈으로 판정한 적이 없다는 것이다.** 생성된 샘플은 [results/phase4](../../PIERROT/results/phase4)에 step 25k~220k 구간 844장이 남아 있지만, 다른 체급처럼 그리드로 묶어 관찰하고 기록한 문서가 없다. 즉 **3B가 1.6B보다 나은지 나쁜지 판단할 근거가 없는 상태로 1차가 끝났다.**

다만 로그에서 읽히는 신호는 조심스럽다. 49,000 step 동안 손실이 1.6174 → 1.6550으로 **내려가지 않았다.** 같은 시기 1.6B가 1.20 수준이었던 것과 대비된다. 이 차이는 세 가지 중 하나일 수 있고, 지금 데이터로는 구분되지 않는다.

- **(a) 정상**: 깊이 성장 직후의 붕괴·회복 구간이 아직 안 끝났다. phase3도 5k에서 무너졌다가 50k에서 돌아왔으므로, 3B는 더 큰 만큼 회복 구간이 길 수 있다.
- **(b) 설정 문제**: phase3에는 있던 REPA(표현 정렬 보조 손실)가 phase4에는 없다. 수렴을 가속하는 장치가 빠진 채로 더 큰 모델을 돌린 셈이다.
- **(c) 라우팅 과다**: TREAD 라우팅 구간이 [2, 48)로 넓어져 매 step 실제로 갱신되는 블록의 비중이 달라졌을 수 있다.

**2차 실험에서 가장 먼저 해야 할 일은 이 세 갈래를 가르는 것이다.** 비용도 가장 싸다 — 남아 있는 305k 체크포인트로 샘플 그리드를 뽑아 보는 것만으로 (a)의 상당 부분이 판정된다. **GPU 1장, 수십 분이면 된다.**

## 4. 외부 모델과의 위치 — PRX 비교

PIERROT의 0.8B 4개 시점과 1.6B 6개 시점, 그리고 PRX(`Photoroom/prx-1024-t2i-beta`, 약 1.17B)를 **같은 프롬프트 20개**로 나란히 생성해 비교했다. 전문은 [vs_prx.md](vs_prx.md)에 있다.

**앞선 것**

- **속성 결합**(누가 무슨 색 옷을 입었는가)이 강하다. 여러 인물에게 각기 다른 색·소지품을 지정한 프롬프트에서 열 체크포인트 모두 정확히 배치했다. PRX는 인물을 뒷모습으로 그려 모호했다.
- **짧은 문구의 글자 쓰기**가 PRX보다 낫다. 다만 시점별로 편차가 크다 — 아래 4.1에서 따로 다룬다.
- **분위기·색감의 밀도**와 **반사 대칭**이 낫다.

**뒤진 것**

- **동작·순간 포착이 약하다.** "공중에 뜬", "드리프트 중인" 같은 요청을 **열 체크포인트가 전부** 무시하고 정적인 장면을 그렸다. 체급·학습량과 무관하게 같은 방향으로 실패했다는 점이 중요하다 — **파라미터로 풀리는 문제가 아니라 데이터 문제**라는 강한 신호다.
- **미세 질감의 사실성**이 부족하다. 정동석 결정이 매끈한 보석처럼, 라멘이 면발 없는 스튜처럼 나온다.
- **프롬프트 충실도**가 떨어진다. 요청하지 않은 것을 넣거나 요청한 것을 빠뜨린다.
- **회화풍 편향**이 남아 있다. 사진을 요청해도 유화 질감이 섞인다.

### 4.1 부수적으로 드러난 것 — 능력은 "획득"되지만 "안정화"되지는 않았다

체크포인트를 열 개까지 촘촘히 늘리자, 원래 목적(외부 모델 대비 위치 확인)과 별개로 **한 능력이 학습 도중 생겨나는 과정**이 통째로 관측됐다. 글자 쓰기(scene text)가 그것이다.

| 체크포인트 | "MORNING LOAF" | "DEEP WORK" | 단계 |
| --- | --- | --- | --- |
| 0.8B 2.00M | `IMORISNNG LOR NNIIIG` | `OO EPPP` | 판독 불가 |
| 0.8B 2.50M | `MORNIING IOODINE` | `DEEPTN` | 판독 불가 |
| 0.8B 2.60M | `MORNING TOORLOFF` | `DEEPEK` | 앞 단어만 |
| 0.8B 2.745M | `MORNING LOAFF` | `DEEP WORRK` | 글자 중복 |
| 1.6B 700k | 정확 | 정확 | **획득** |
| 1.6B 750k | 정확 | `DEEPP WORK` | 흔들림 |
| 1.6B 800k | 정확 | 정확 | 정확 |
| 1.6B 850k | 정확 | 정확 | 정확 |
| 1.6B 900k | 정확 | `DEEEP WORK` | 흔들림 |
| 1.6B 930k | `MORING LOAF` | 정확 | 흔들림 |

읽어야 할 것은 두 가지다.

**첫째, 능력이 단계를 밟아 올라온다.** 판독 불가 → 앞 단어만 맞음 → 글자 중복 → 정확. 특히 2.60M의 `MORNING TOORLOFF`는 **긴 문자열의 앞쪽부터 자리를 잡는다**는 것을 보여준다. 따옴표 안 텍스트를 글자 단위로 토큰화하는 처리가 학습량이 쌓이며 실제로 작동하기 시작한 것으로 보인다.

**둘째, 획득 이후에도 계속 흔들린다.** 700k에서 두 문구 모두 정확해졌지만 750k·900k·930k에서 산발적으로 틀린다. 틀리는 방식도 일정하지 않다(`DEEPP` P중복, `DEEEP` E중복, `MORING` N누락). 그리고 **두 문구가 동시에 틀리지는 않는다** — 특정 시점이 "글자를 못 쓰게 되는" 것이 아니라 매번 다른 지점에서 미끄러진다.

**이것이 방법론에 주는 교훈이 더 크다.** 처음에 800k·850k 두 시점만 봤을 때는 "능력이 자리 잡았다"고 결론 냈으나, 앞뒤(700k·750k·900k)를 채우자 **아직 안정화되지 않은 상태**임이 드러났다. 성긴 간격으로 뽑은 체크포인트는 능력의 획득 시점을 늦게 잡고, 흔들림을 통째로 놓친다.

같은 위험이 7절에서 지적한 정성 평가의 약점과 겹친다. **2차 실험에서는 "언제 생겼는가"와 "안정적인가"를 나눠 측정해야 한다** — 글자 수를 4·8·12·16자로 늘려 가며 여러 시점에서 반복 측정하는 식의 설계가 필요하다.

**이 비교는 공정한 벤치마크가 아니다.** PIERROT는 base 학습이 진행 중인 중간 체크포인트이고 후처리 학습을 거치지 않았으며, PRX는 완성되어 배포된 모델이다. 텍스트 인코더 구성도 다르다(PIERROT는 Qwen3-4B 중간 3개 레이어 concat, PRX는 T5Gemma 2.61B). 샘플은 프롬프트당 1장·seed 1개다. 자세한 한계는 [vs_prx.md](vs_prx.md) 4절에 정리돼 있다.

## 5. 반복해서 나타난 하나의 현상 — 회화풍 편향

1차 실험 전체에서 **가장 일관되게 관측된 문제**이므로 따로 다룬다.

"회화풍(painterly) 편향"이란, 사진을 요청했는데도 붓질 질감·과장된 명암·유화 같은 색층이 나타나는 현상이다. 프롬프트가 시키지 않았는데 모델이 스스로 그 방향으로 간다.

**두 체급에서 같은 모양의 곡선이 반복됐다.**

| 모델 | 깨끗한 구간 | 회화풍 정점 | 이후 |
| --- | --- | --- | --- |
| 0.8B (phase2) | 1.0M | 2.0M | 2.5M에서 회복 |
| 1.6B (phase3) | 300k | 500k ~ 625k | 655k 이후 안정 구간 유지 |

**모델 크기가 달라도 같은 곡선을 그린다는 사실이 결정적이다.** 구조가 원인이라면 레이어 16장 모델과 33장 모델이 같은 모양으로 흔들릴 이유가 없다. 두 트랙이 공유하는 것은 **학습 데이터의 분포**뿐이다.

한 가지 비유로 설명하면 이렇다. 학습 데이터는 여러 출처를 섞은 것이고, 그중 일부(합성 이미지·아트 계열)는 회화적 스타일이 강하다. 모델은 처음에는 "무엇을 그릴지"를 배우느라 스타일까지 신경 쓸 여유가 없다. 어느 정도 형태를 익히고 나면, 남은 오차를 줄이는 가장 손쉬운 길이 **데이터에서 가장 자주 보이는 스타일로 전부 수렴시키는 것**이 된다. 그래서 중반에 스타일이 한쪽으로 쏠렸다가, 학습이 더 진행되며 프롬프트별 구분이 세밀해지면 다시 풀린다.

실제로 데이터 쪽 증거도 있다. 데이터셋별 이미지 품질을 자동 채점한 결과, 상위는 `flux_reason_6m`(77.00)·`flux_generated`(72.12) 같은 **합성 데이터**였다. 그리고 1차 실험 중에 두 번, 데이터 분포 때문에 학습이 실제로 망가진 사건이 있었다.

- **midjourney 캡션 교체 사고**: step 약 1,490,000에서 캡션 약 1.8%가 짧은 형식에서 595자 장문으로 자동 교체되며 픽셀 변화량이 ±5에서 ±20으로 튀었다. 체크포인트 2개를 버리고 step 1,480,000에서 재개했다.
- **`blip3o_60k` 캡션 오염**: 회화·CGI 이미지의 캡션에 "realistic"이 들어 있어, `realistic`이라는 단어가 **르네상스 회화**를 부르는 방아쇠가 되어 있었다. 캡션 1,499개를 고쳐 잡았다.

> **2차 실험에 주는 결론**: 회화풍 편향은 모델을 키우거나 더 오래 돌려서 푸는 문제가 아니다. **데이터 혼합 비율과 캡션 품질이 손댈 자리다.**

## 6. 자원 회계

1차 실험이 실제로 쓴 것을 남겨 둔다. 2차 계획의 현실성 판단에 필요하다.

| 항목 | 값 |
| --- | --- |
| 하드웨어 | NVIDIA B200 192GB × 8, 노드 2대 (looko1 / looko2), 50GbE 직결 |
| 학습 기간 (관측) | 2026-05-15 ~ 2026-07-20, 약 66일 |
| 저장소 | Lustre 20 TB |
| 체크포인트 총량 | **약 3.6 TB** (phase3 1.3 TB / phase4 907 GB / phase2 905 GB / phase1 115 GB / SFT 337 GB) |
| 크래시 | **0건** — 다섯 로그 전부 OOM·NCCL timeout·NaN·traceback 없음 |

**크래시 0건은 그 자체로 성과다.** 1차 초반에는 워커 프로세스 강제 종료, 손상된 이미지 1장이 배치 전체를 죽이는 문제, GPU 메모리 파편화 등이 반복됐고, 그것들을 하나씩 잡은 결과가 이 숫자다. 안정화에 쓴 시간이 회수됐다는 뜻이다.

**반면 회수되지 않은 비용도 분명하다.**

- phase1 KD: 115 GB + 약 2.4일 → 최종 계보에 기여 0
- phase2_sft 1차: 167 GB 삭제 (보조 손실을 끄고 돌린 것)
- 초기 체크포인트 삭제로 0.5M 이전 구간은 **되돌아가 관찰할 수 없다**

마지막 항목은 2차에서 반드시 고칠 부분이다. 그리드 한 장 뽑는 데는 몇 분이면 되지만, **지운 체크포인트는 되돌릴 수 없다.** 체크포인트를 지우기 전에 관찰 그리드를 먼저 뽑아 두는 규칙이 필요하다.

## 7. 측정하지 못한 것 — 정직하게

1차 실험의 결론은 **거의 전부 사람 눈으로 본 정성 관찰**이다. 이것이 가장 큰 구조적 약점이다.

| 지표 | 상태 |
| --- | --- |
| GenEval | 🔶 단 1회 (phase2 step 2.695M, 0.4416) — 그마저 채점 파이프라인 문제 의심 |
| FID | ❌ 없음 (설정에서 비활성) |
| CLIP score | ❌ 없음 |
| DPG | ❌ 없음 |
| phase3 · phase4 수치 평가 | ❌ 없음 |
| 사람 blind A/B | ❌ 없음 |

계획 문서에는 FID-30k, CLIPScore, LPIPS 다양성, 200 프롬프트 blind A/B까지 설계되어 있었지만 **하나도 실행되지 않았다.** 그래서 "1.6B가 0.8B보다 낫다", "3B가 어떻다" 같은 문장을 숫자로 뒷받침할 수 없다.

정성 관찰이 무가치하다는 뜻은 아니다. 실제로 회화풍 편향은 손실 곡선으로는 전혀 보이지 않았고 눈으로만 잡혔다. **둘 다 필요한데 한쪽만 있었다**는 것이 문제다.

여기에 더해, **체크포인트를 얼마나 촘촘히 관찰했는가**도 결론의 질을 좌우했다. 4.1절에서 보듯 두 시점(800k·850k)만 봤을 때와 여섯 시점(700k~930k)을 봤을 때의 결론이 정반대였다 — 전자는 "능력 확보", 후자는 "아직 흔들리는 중"이었다. **지표를 늘리는 것만큼 관찰 간격을 좁히는 것도 중요하다.**

## 8. 2차 실험 계획의 근거

위 내용을 우선순위로 옮기면 다음과 같다. 순서는 **비용 대비 정보량**으로 매겼다.

| 순위 | 할 일 | 비용 | 왜 먼저인가 |
| --- | --- | --- | --- |
| 1 | **phase4 3B 305k 체크포인트 샘플 그리드** | GPU 1장 × 수십 분 | 1차의 가장 큰 미지수를 가장 싸게 없앤다. 학습 재개 없이 지금 당장 가능 |
| 2 | **GenEval 재평가** (chi_prompt 조건 일치 + 검출기 점검) | GPU 1장 × 수 시간 | 유일한 수치가 신뢰 불가 상태다. 0.44가 진짜인지부터 확정 |
| 3 | **FID / CLIP score 파이프라인 완성** | 학습 불필요 | 2차의 모든 판단을 숫자로 뒷받침하기 위한 전제 조건 |
| 4 | **데이터 혼합 재설계** (회화풍·합성 비중, 동작 캡션 보강) | 학습 재개 필요 | 5절·4절에서 확인된 **두 개의 근본 원인**을 동시에 겨냥 |
| 5 | **phase4 설정 교정** (REPA 재도입, 라우팅 구간 재검토) | 학습 재개 필요 | 3.5절 (b)(c) 가설의 검증. 1번 결과에 따라 필요 없어질 수도 있음 |
| 6 | **SFT 재시도** (보조 손실 ON, 실사 비중 확보) | 학습 재개 필요 | 원인 가설은 이미 있다. 검증만 남았다 |
| 7 | **능력별 획득·안정성 추적 세트** (글자 길이·개수·동작 축) | GPU 소량, 학습 불필요 | 4.1절에서 확인된 "획득 후 흔들림"을 상시 관측 가능하게 만든다 |

**1~3번은 GPU가 거의 없어도 된다.** 학습을 다시 돌리는 것이 아니라 이미 있는 체크포인트를 평가하는 일이기 때문이다. GPU 확보를 기다리는 동안 할 수 있는 작업이고, 이걸 해 두면 4~6번의 설계가 추측이 아니라 근거 위에서 이뤄진다.

### 2차에 넘길 확정된 결론 세 가지

1. **깊이 성장은 작동한다.** 0.8B → 1.6B에서 5만 step 만에 회복했다. 3B 확장의 성패는 이 방식 자체가 아니라 개별 설정에 달렸다고 봐야 한다.
2. **회화풍 편향과 동작 표현 실패의 원인은 데이터 쪽이다.** 두 체급이 같은 곡선을 그렸고, 열 체크포인트가 같은 방향으로 실패했다. 모델을 키워서 풀 문제가 아니다.
3. **정성 관찰만으로는 부족하다.** 수치 파이프라인을 먼저 세우지 않으면 2차도 같은 자리에서 끝난다.
4. **능력은 한 번 생기면 유지되는 것이 아니다.** 글자 쓰기는 700k에서 획득됐지만 이후 시점에서 계속 미끄러졌다(4.1절). 특정 체크포인트 한둘로 "된다/안 된다"를 판정하면 틀린다. **여러 시점에서 반복 확인**하는 것을 기본 절차로 삼아야 한다.
